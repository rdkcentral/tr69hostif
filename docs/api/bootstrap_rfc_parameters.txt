# WHAT IS THE BOOTSTRAP STORE?
# =============================
# The Bootstrap store (XBSStore) is a separate key-value store from the main
# RFC store. It is backed by:
#   Runtime file:  /opt/secure/RFC/bootstrap.ini
#   Defaults file: /etc/partners_defaults.json  (partner-keyed JSON)
#   Partner ID:    /opt/www/authService/partnerId3.dat
#
# At daemon start, XBSStore::loadFromJson() reads partners_defaults.json for
# the active partner and populates the in-memory store.  Individual parameters
# can be overridden at runtime by RFC push or WebPA set.
#
# ROUTING MECHANISM
# =================
# Parameters are routed to the Bootstrap store when the data-model XML marks
# them with bsUpdate="allUpdate".  The request handler
# (hostIf_DeviceClient_ReqHandler.cpp) checks bsUpdate != HOSTIF_NONE and
# diverts GET/SET to XBSStore::getValue() / XBSStore::overrideValue() instead
# of the RFC store.
#
# WRITE ACCESS CONTROL
# ====================
# SET is only accepted from:
#   - HOSTIF_SRC_RFC   (server-side RFC push)
#   - HOSTIF_SRC_WEBPA (WebPA/parodus set)
# All other requestors are silently rejected.
#
# SOURCE PRIORITY
# ===============
# BSStore journal tracks which source last wrote each key.
# A HOSTIF_SRC_DEFAULT (JSON) write will NOT overwrite an active RFC/WebPA value.
#
# Implementation sources:
#   src/hostif/profiles/DeviceInfo/XrdkCentralComBSStore.cpp  (store impl)
#   src/hostif/profiles/DeviceInfo/XrdkCentralComBSStore.h    (class declaration)
#   src/hostif/profiles/DeviceInfo/Device_DeviceInfo.cpp       (get/set handlers)
#   src/hostif/handlers/src/hostIf_DeviceClient_ReqHandler.cpp (bsUpdate routing gate)
#   src/hostif/parodusClient/waldb/data-model/data-model-generic.xml (bsUpdate="allUpdate")
#   src/hostif/parodusClient/waldb/data-model/data-model-stb.xml     (STB-platform bsUpdate)
#   partners_defaults.json                                           (partner defaults)
#   src/integrationtest/conf/bootstrap.ini                           (integration test seed)
#
# Legend:
#   RO  = Read-only (cannot be SET via WebPA/RFC)
#   RW  = Read-write (RFC/WebPA only)
